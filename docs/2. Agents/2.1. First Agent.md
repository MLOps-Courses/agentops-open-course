# 2.1. First Agent

## What are you building first?

You are assembling the smallest useful slice of the final Ops Copilot: typed settings, an explicit model selector, a clear instruction, read-only incident tools, and a runnable ADK root agent. Later pages explain each capability; this page shows how the pieces compose.

## How is configuration parsed?

`pydantic-settings` validates the `AGENT_*` boundary once:

```python
class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
        env_prefix="AGENT_",
        extra="ignore",
    )

    model: str = Field(default="gemini-3.5-flash", min_length=1)
    gateway_enabled: bool = False
    mcp_url: str | None = None
    data_dir: Path = _DEFAULT_DATA_DIR
    state_dir: Path = _DEFAULT_STATE_DIR
    a2a_host: str = Field(default="localhost", min_length=1)
    a2a_port: int = Field(default=8080, ge=1, le=65535)
    a2a_protocol: str = Field(default="http", pattern=r"^https?$")
    a2a_max_llm_calls: int = Field(default=12, ge=1, le=100)
```

This separates immutable seed data from writable state, separates the A2A bind address from the address advertised to clients, and bounds the model loop for one network request.

## How is the model selected?

The application has two explicit branches:

```python
def build_model() -> str | BaseLlm:
    if not settings.gateway_enabled:
        return settings.model
    if not os.environ.get("OPENAI_BASE_URL"):
        raise ValueError("AGENT_GATEWAY_ENABLED=true requires OPENAI_BASE_URL")
    if not os.environ.get("OPENAI_API_KEY"):
        raise ValueError("AGENT_GATEWAY_ENABLED=true requires OPENAI_API_KEY")
    return OpenAILlm(model=settings.model)
```

Native mode lets ADK resolve the explicit Gemini model name. Gateway mode uses ADK's OpenAI-compatible adapter and fails if the endpoint contract is incomplete.

## How is the root agent defined?

```python
root_agent = Agent(
    model=build_model(),
    name="agentops_agent",
    description="An on-call Ops Copilot that triages and resolves incidents from a local dataset.",
    instruction=INSTRUCTION,
    tools=[*_read_tools(), *ACTION_TOOLS, skill_toolset()],
    before_model_callback=redact_request_pii,
    after_model_callback=redact_response_pii,
    before_tool_callback=validate_actions,
    after_tool_callback=redact_tool_output_pii,
    on_model_error_callback=handle_model_error,
    on_tool_error_callback=handle_tool_error,
)
```

The composition is intentionally visible. `_read_tools()` returns direct read/knowledge functions locally or one governed MCP toolset when `AGENT_MCP_URL` is configured. Writes and skill loading keep separate owners; policy is attached at the runtime boundary rather than hidden inside the prompt.

## How does the ADK CLI discover it?

`agents/python/src/agent/__init__.py` imports `root_agent`. The `adk` CLI receives the package path and discovers that conventional name:

```python
from .agent import root_agent

__all__ = ["root_agent"]
```

Do not put network calls or destructive setup in package import code. Discovery, tests, and tooling all import this module.

## How do you run it with native Gemini?

From `agents/python/`, after setting `GOOGLE_API_KEY` in the repository-root `.env`:

```bash
mise run web
# or
mise run run
```

Start with `List the open incidents`. The expected behavior is a `list_incidents` tool call followed by a grounded summary. Wording can differ; invented incidents or skipped tools are failures.

## Can you verify the agent without a model?

Yes. Construction, configuration errors, tools, callbacks, MCP/A2A application wiring, and state are covered offline:

```bash
mise run test
```

This is the default checkpoint. Interactive output is useful exploration, not proof that all boundaries work.

## What is the page checkpoint?

Run the tests, then inspect `root_agent.tools` in a Python shell or test. Confirm that read, knowledge, action, and skill toolsets are registered and that gateway mode rejects a missing base URL/key. Continue when both branches are intentional rather than environment-dependent surprises.

# 2.2. Models

The model is the reasoning engine at the center of the agentic loop. This section shows how the Ops Copilot is wired to a native **Gemini** model, how to authenticate, and why every other provider is deferred to the gateway in Chapter 5.

## Which model does the agent use?

The default is **`gemini-3.5-flash`** — Google's fast, low-cost model, plenty for triaging incidents over a small dataset. The agent sets it **explicitly** rather than relying on an ADK default, because those defaults change between releases and would make behavior irreproducible.

```python
# agents/python/src/agent/config.py
class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8", env_prefix="AGENT_", extra="ignore")

    # Set an explicit model — ADK defaults churn. Override with AGENT_MODEL.
    model: str = "gemini-3.5-flash"
```

## How is the model wired to the agent?

A model identifier is a string; ADK turns it into a live client. You pass the string straight to `Agent(model=...)` and ADK constructs the Gemini client for you.

```python
# agents/python/src/agent/agent.py
from google.adk import Agent

from .config import settings

root_agent = Agent(
    model=settings.model,  # "gemini-3.5-flash" — ADK builds the Gemini client
    name="agentops_agent",
    # ...
)
```

You end up with a native Gemini client — no wrapper, no LiteLLM.

## How do you authenticate to Gemini?

The SDK reads its environment variables (loaded from the repo-root `.env` by mise), so you configure credentials once. There are two supported paths:

1. **AI Studio API key** — a direct key (`AIza…`) from [ai.google.dev](https://ai.google.dev), talking to the `generativelanguage` endpoint. Simplest to start.
1. **Vertex AI** — route through Vertex, driven by `GOOGLE_GENAI_USE_VERTEXAI=true`. This is required for a Vertex AI **express** key, and it is also the path for Application Default Credentials (ADC) and project-based access.

```bash
# .env (repo root; gitignored)
GOOGLE_API_KEY=your-key-here
AGENT_MODEL=gemini-3.5-flash

# Route to Vertex AI. Required for a Vertex AI express key; also enables ADC + project mode.
# GOOGLE_GENAI_USE_VERTEXAI=true
```

## What does `GOOGLE_GENAI_USE_VERTEXAI` actually switch?

It picks which Google backend the underlying `google-genai` client talks to:

- **Unset or `false`** → the direct Gemini API (AI Studio). Uses `GOOGLE_API_KEY` as an AI Studio key.
- **`true`** → Vertex AI. Uses a Vertex AI express key, or Application Default Credentials with a project and location.

!!! warning "This project's key is a Vertex AI express key"

    The course's provided key is a **Vertex AI express** key. Against the direct Gemini API it returns `403 API_KEY_SERVICE_BLOCKED`; you must set `GOOGLE_GENAI_USE_VERTEXAI=true` for it to work. If your own key is a plain AI Studio key, leave that variable unset. Either way, set `model=` explicitly — defaults churn.

## How do you change the model?

Override `AGENT_MODEL` in `.env` (or the environment); no code change. The agent honors it via the `AGENT_` settings prefix. Swap in `gemini-3.5-pro` for harder reasoning, or a lighter Gemini for cost, and re-run.

## What about OpenAI, Anthropic, or Ollama?

Not in this chapter. Chapters 2 through 4 stay on **native Gemini** deliberately: it is the simplest, most idiomatic ADK setup.

True vendor-neutrality — OpenAI, Anthropic, OpenRouter, local **Ollama**, and more — arrives in [Chapter 5](../5. Gateway/) by pointing the agent's model endpoint at **agentgateway's LLM backend**. That is one endpoint switch, zero SDK changes.

!!! info "No LiteLLM"

    A common shortcut is to route providers through LiteLLM. This course **intentionally does not** — the gateway is the single, vendor-neutral seam for every provider, so there is one place to secure, observe, and rate-limit model traffic. See [0.4. Providers](../0. Overview/0.4. Providers.md) for the full rationale.

# 2.2. Models

## Why is the model explicit?

Agent behavior, latency, context limits, tool use, and cost can change between model versions. The repository declares `AGENT_MODEL` with a concrete default instead of relying on a provider alias or SDK default:

```python
model: str = Field(default="gemini-3.5-flash", min_length=1)
```

The lockfile can pin an SDK, but it cannot pin a hosted provider's weights. Record the model name, provider path, prompt version, and evaluation result together.

## How does native Gemini work?

When gateway mode is false, `build_model()` returns the configured string. ADK's native Gemini integration owns the provider client:

```python
if not settings.gateway_enabled:
    return settings.model
```

Set a Gemini API key in the gitignored root `.env`, or use the supported Vertex/ADC configuration. The application does not inspect key prefixes or silently switch products.

## How does the gateway model work?

When `AGENT_GATEWAY_ENABLED=true`, the builder returns ADK's `OpenAILlm`. The OSS OpenAI client reads its standard endpoint variables:

```bash
AGENT_GATEWAY_ENABLED=true
AGENT_MODEL=qwen3:4b
OPENAI_BASE_URL=http://localhost:4000/v1
OPENAI_API_KEY=agentgateway
```

agentgateway exposes one OpenAI-compatible application contract and selects Ollama/Qwen3 locally or Vertex/Gemini on GKE. No provider-specific SDK is added to the agent for that switch.

## Why require an API key for a local model?

The OpenAI client requires a non-empty key even when the local gateway route has no authentication. `agentgateway` is a marker, not a secret. In a real shared environment, replace that marker with actual client authentication and keep upstream cloud identity at the gateway.

## Does the course use LiteLLM?

No. Native Gemini uses ADK directly; gateway mode uses ADK's OpenAI-compatible adapter. The application, evaluation, and deployment paths have no LiteLLM dependency.

## How should you compare models?

Hold the prompt, seed data, tool schemas, and evaluation cases constant. Compare at least:

- Exact tool trajectory success.
- Grounding and hallucination failures.
- End-to-end latency and model-call count.
- Input/output tokens and provider price where applicable.
- Policy refusal and adversarial regression behavior.

A model that writes nicer prose but chooses the wrong tool is not an upgrade for this agent.

## What is the model checkpoint?

Run the model configuration tests, then use one read-only prompt on your chosen path:

```bash
cd agents/python
uv run pytest tests/test_model.py
mise run run
```

Record the selected model and endpoint path. Do not compare two models from different runtime state; reset with `mise run data:reset` first.

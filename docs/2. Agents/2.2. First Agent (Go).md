# 2.2. First Agent (Go)

_This is a **Go** track page. Building in Python instead? Skip to [2.1. First Agent (Python)](./2.1. First Agent (Python).md)._

This is the hero build. By the end you have the reference agent — the **Ops Copilot** — running in both a browser UI and your terminal, compiled to a single binary. It assumes you finished [Chapter 1](../1. Setup/) (mise, the Go toolchain, and a provider key in `.env`).

## What are you building?

The **Ops Copilot** is an on-call assistant that helps engineers triage and resolve incidents for a fictional platform, using a 100% local, bundled dataset — no external services. In this chapter it already carries its tools; you build the agent that ties a Gemini model, an on-call persona, and those tools together, then run it.

The Go track lives in `agents/go/` (module `github.com/MLOps-Courses/agentops-open-course/agents/go`), built on the ADK library at `google.golang.org/adk/v2`. Its layout:

- `internal/config/config.go` — typed configuration (which model, which key, where the data is).
- `internal/agent/agent.go` — the agent constructor (`New`).
- `cmd/agent/main.go` — the entry point that launches it.

## How does the agent find its configuration?

Configuration is a plain struct filled from the environment, with explicit defaults so nothing is left implicit:

```go
// agents/go/internal/config/config.go
// DefaultModel is set explicitly because ADK model defaults churn.
// Native Gemini in Ch. 2-4; other providers arrive via agentgateway in Ch. 5.
const DefaultModel = "gemini-3.5-flash"

// Config is the agent configuration.
type Config struct {
	Model   string // LLM identifier.
	APIKey  string // GOOGLE_API_KEY for the native Gemini API.
	DataDir string // Directory holding the bundled Ops Copilot dataset (incidents.db, runbooks/).
}

// Load reads configuration from the environment, applying defaults.
func Load() Config {
	model := os.Getenv("AGENT_MODEL")
	if model == "" {
		model = DefaultModel
	}
	return Config{
		Model:   model,
		APIKey:  os.Getenv("GOOGLE_API_KEY"),
		DataDir: dataDir(),
	}
}
```

The **model is set explicitly** (`gemini-3.5-flash`): ADK's built-in defaults change between releases, so pinning it keeps behavior reproducible. `AGENT_MODEL` overrides it, and `GOOGLE_API_KEY` supplies the credential wired up in [2.3. Models](./2.3. Models.md).

## How do you define the agent?

The constructor `New` builds a Gemini model, opens the dataset, gathers the tools, and hands them to `llmagent.New`:

```go
// agents/go/internal/agent/agent.go
// New builds the reference agent on a native Gemini model, wired to the bundled dataset.
// The caller owns the returned data.Store and should Close it on shutdown.
func New(ctx context.Context, cfg config.Config) (adkagent.Agent, *data.Store, error) {
	model, err := gemini.NewModel(ctx, cfg.Model, &genai.ClientConfig{APIKey: cfg.APIKey})
	if err != nil {
		return nil, nil, fmt.Errorf("creating gemini model %q: %w", cfg.Model, err)
	}

	store, err := data.Open(cfg.DataDir)
	if err != nil {
		return nil, nil, fmt.Errorf("opening dataset: %w", err)
	}

	// ... build toolset, knowledge, and actionTools from the store ...

	a, err := llmagent.New(llmagent.Config{
		Name:                Name,
		Model:               model,
		Description:         Description,
		Instruction:         Instruction,
		Tools:               allTools,
		BeforeToolCallbacks: []llmagent.BeforeToolCallback{guardrails.ValidateActions},
	})
	if err != nil {
		_ = store.Close()
		return nil, nil, fmt.Errorf("building agent: %w", err)
	}
	return a, store, nil
}
```

`llmagent.New(llmagent.Config{...})` is the Go equivalent of the `LlmAgent` from [2.0. Concepts](./2.0. Concepts.md). Its fields map onto the concepts you know:

- `Model` — the Gemini model from `gemini.NewModel`, detailed in [2.3. Models](./2.3. Models.md).
- `Name` and `Description` — the agent's identity, used by the UI and by other agents over A2A.
- `Instruction` — the persona and operating rules (below), detailed in [2.4. Instructions](./2.4. Instructions.md).
- `Tools` — the incident, knowledge, and action tools the model may call, built in [Chapter 3](../3. Capabilities/).
- `BeforeToolCallbacks` — guardrails that run before any tool, so state-changing actions need approval. Covered in [4.5. Guardrails](../4. Quality/4.5. Guardrails.md).

Note the signature: `New` returns the agent **and** a `*data.Store`. The caller owns that store and must `Close` it on shutdown — a fail-fast, explicit-ownership pattern idiomatic to Go. The identity constants come from the same file:

```go
// agents/go/internal/agent/agent.go
// Agent identity, shared with other agents (A2A) and the developer UI.
const (
	Name        = "agentops_agent"
	Description = "An on-call Ops Copilot that triages and resolves incidents from a local dataset."
)
```

!!! note "The tools are already written"

    `tools.All`, `memory.KnowledgeTools`, and `actions.All` come from other packages under `internal/`. You do not need to understand them yet — Chapter 3 builds each one. For now they make the agent genuinely useful so you can watch the loop call them.

## How is the agent launched?

`main.go` builds the agent and hands it to ADK's `full` launcher, which exposes every run mode — `console`, plus the `web` server and its sub-servers `webui` (the browser UI), `api` (REST), and `a2a` — selected by the command-line arguments:

```go
// agents/go/cmd/agent/main.go
func main() {
	ctx := context.Background()

	a, store, err := agent.New(ctx, config.Load())
	if err != nil {
		log.Fatalf("failed to build agent: %v", err)
	}
	defer func() { _ = store.Close() }()

	cfg := &launcher.Config{AgentLoader: adkagent.NewSingleLoader(a)}

	l := full.NewLauncher()
	if err := l.Execute(ctx, cfg, os.Args[1:]); err != nil {
		log.Fatalf("run failed: %v\n\n%s", err, l.CommandLineSyntax())
	}
}
```

Unlike Python's standalone `adk` CLI, Go has no separate binary: your `main` _is_ the launcher. `full.NewLauncher()` bundles all modes, and `l.Execute(ctx, cfg, os.Args[1:])` dispatches on the command-line arguments. The two top-level keywords are `console` and `web`; the `web` server needs you to name at least one sub-server (`webui`, `api`, or `a2a`) — running `web` alone errors with the list of valid sub-servers. The `defer store.Close()` guarantees the dataset is released on exit.

## How do you run it?

The `run` task passes its arguments straight through to the launcher, so you pick the mode after `--`:

```bash
cd agents/go
mise run run -- web webui    # ADK web UI (browser)
mise run run -- console      # terminal REPL
```

The `full` launcher's run modes are `console` (the default), `web webui` (browser UI), `web api` (REST server), and `web a2a` (the A2A endpoint). `web` on its own errors, because it needs a sub-server named after it.

`mise run run` wraps `go run ./cmd/agent` (see `agents/go/mise.toml`). Open the web UI's printed URL, pick the `agentops_agent`, and chat. Try:

> _List the open incidents, then tell me what's wrong with checkout._

The model calls `list_incidents`, then `get_service_status`, then answers from the results — the agentic loop from [2.0. Concepts](./2.0. Concepts.md), running live. The `console` mode gives the same agent as a terminal REPL. The full set of modes is the subject of [2.6. Dev Loop](./2.6. Dev Loop.md).

!!! warning "You need a provider key"

    Every run mode calls a real Gemini model, so it needs `GOOGLE_API_KEY` in `.env`. If you skipped it, set that up in [1.5. Providers](../1. Setup/1.5. Providers.md) and [2.3. Models](./2.3. Models.md) first.

## How do you build the binary?

For a standalone artifact — the single-binary advantage of Go — compile it:

```bash
mise run build      # go build -o bin/agent ./cmd/agent
```

That produces `bin/agent`, which you can run directly (`./bin/agent web webui`) with no toolchain present. This is the same binary Chapter 6 packages into a container image for Kubernetes.

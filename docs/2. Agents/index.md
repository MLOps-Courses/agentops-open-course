# 2. Agents

This is where you build your first agent — the **Ops Copilot**, the reference agent carried through the whole course. You start from the concepts, build and run the full agent in your language, then take one piece at a time: the model, the instruction, sessions, and the developer loop.

Read [2.0. Concepts](./2.0. Concepts.md) first, then your track's First Agent page, then work through the shared tabbed sections (pick your language once and it follows you site-wide):

- **[2.0. Concepts](./2.0. Concepts.md)**: The ADK 2.0 building blocks — Agent, Runner, Session, Events, Tools, and the graph Workflow.
- **[2.1. First Agent (Python)](./2.1. First Agent (Python).md)**: Build and run the Ops Copilot end to end in Python _(Python track)_.
- **[2.2. First Agent (Go)](./2.2. First Agent (Go).md)**: Build and run the Ops Copilot end to end in Go _(Go track)_.
- **[2.3. Models](./2.3. Models.md)**: Wire a native Gemini model, choose API-key or Vertex AI auth, and defer other providers to the gateway.
- **[2.4. Instructions](./2.4. Instructions.md)**: The system instruction — persona, operating rules, grounding, and structured output.
- **[2.5. Sessions](./2.5. Sessions.md)**: Sessions, state, and short-term memory, with the minimal programmatic run pattern.
- **[2.6. Dev Loop](./2.6. Dev Loop.md)**: Running the agent locally — web UI, terminal REPL, and API server in both tracks.

By the end you have a working, Gemini-backed agent you can run and observe. [Chapter 3](../3. Capabilities/) gives it tools, memory, workflows, and delegation.

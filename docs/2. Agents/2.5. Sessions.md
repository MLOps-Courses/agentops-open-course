# 2.5. Sessions

In [2.1](./2.1. First Agent (Python).md) / [2.2](./2.2. First Agent (Go).md) the `adk` CLI and the Go launcher ran the agent for you and managed sessions behind the scenes. This section opens that box: what a session is, how the Runner uses it, and how to drive the agent yourself in a few lines — the pattern you reach for in tests and scripts.

## What is a session?

A **session** is one conversation with the agent: a stable id, tied to a user, holding the ordered list of events that have happened so far. It is the agent's **short-term memory** — the context the model sees on the next turn. When you send a follow-up like "and restart it", the agent knows what "it" is because the earlier turns are in the session.

Alongside the events, each session carries **state**: a key/value bag the agent and its tools read and write during a run. State is scratch space for one conversation — a value passed between workflow steps, a flag a tool set. It lives and dies with the session.

!!! info "Sessions are short-term, not long-term memory"

    A session remembers _this_ conversation. Remembering across conversations — past incidents, the runbook knowledge base — is **long-term memory**, a separate service the Ops Copilot gains in [3.4. Memory](../3. Capabilities/3.4. Memory.md). This page is only about the short-term kind.

## Where are sessions stored?

Behind a **session service**. ADK ships an in-memory one — `InMemorySessionService` in Python, `session.InMemoryService()` in Go — that keeps everything in process. It is perfect for development, tests, and this chapter: zero setup, nothing to clean up. It is also **ephemeral**: restart the process and every session is gone. Chapter 6 swaps in a persistent service for deployment; the agent code does not change, only the wiring.

## What does the Runner do with a session?

The **Runner** is the engine that executes the agentic loop from [2.0. Concepts](./2.0. Concepts.md). You hand it an agent and a session service; then, per turn, you give it a user id, a session id, and a new message. It appends your message to the session, calls the model, runs any tools, appends every result back to the session, and yields the whole thing as a stream of events.

So the session is the Runner's working memory, and the events it yields are both the agent's output and the session's growing history.

## How do you run the agent programmatically?

You do not need the CLI to run an agent — a Runner and an in-memory session service are enough. This is the minimal pattern the tests and scripts use: create a runner, create a session, send a message, iterate the events.

=== "Python"

    ```python
    import asyncio

    from google.adk.runners import InMemoryRunner
    from google.genai import types

    from agent.agent import root_agent


    async def main() -> None:
        # InMemoryRunner bundles in-memory session, artifact, and memory services.
        runner = InMemoryRunner(agent=root_agent, app_name="ops-copilot")
        session = await runner.session_service.create_session(app_name="ops-copilot", user_id="engineer")

        message = types.Content(role="user", parts=[types.Part(text="List the open incidents.")])
        async for event in runner.run_async(
            user_id="engineer", session_id=session.id, new_message=message
        ):
            if event.is_final_response() and event.content:
                print(event.content.parts[0].text)


    asyncio.run(main())
    ```

=== "Go"

    ```go
    // The ADK agent package is imported as `adkagent` to avoid colliding with the
    // course's own internal `agent` package (as cmd/agent/main.go does).
    ctx := context.Background()

    a, store, err := agent.New(ctx, config.Load())
    if err != nil {
        log.Fatalf("build agent: %v", err)
    }
    defer func() { _ = store.Close() }()

    sessionService := session.InMemoryService()
    sess, err := sessionService.Create(ctx, &session.CreateRequest{AppName: "ops-copilot", UserID: "engineer"})
    if err != nil {
        log.Fatalf("create session: %v", err)
    }

    r, err := runner.New(runner.Config{AppName: "ops-copilot", Agent: a, SessionService: sessionService})
    if err != nil {
        log.Fatalf("create runner: %v", err)
    }

    message := genai.NewContentFromText("List the open incidents.", genai.RoleUser)
    for event, err := range r.Run(ctx, "engineer", sess.Session.ID(), message, adkagent.RunConfig{}) {
        if err != nil {
            log.Fatalf("run: %v", err)
        }
        if event.Content != nil {
            for _, part := range event.Content.Parts {
                fmt.Print(part.Text)
            }
        }
    }
    ```

The shapes mirror each other. Python's `InMemoryRunner` is a convenience that bundles the in-memory session, artifact, and memory services in one object; in Go you construct the `session.InMemoryService()` and pass it to `runner.New` explicitly — the same trade-off between hidden and explicit wiring you saw with the model in [2.3. Models](./2.3. Models.md).

## How is a message shaped?

A turn's input is a **content** object with a `role` (here, `user`) and one or more **parts** (here, a single text part). ADK uses the same shape for the model's replies, so the events streaming back are contents too — that is why reading a response means walking its parts. The message and the resulting events are all appended to the session, becoming the history the next turn sees.

## How do you keep a conversation going?

Call the Runner again with the **same `session_id`**. Because the session already holds the previous turns, the agent has the full context — send "now restart checkout" after "what's wrong with checkout?" and it resolves the reference. Use a **new** session id to start fresh with no memory of what came before.

This is exactly what the `adk` web UI and the Go console do for you: one session per chat, reused turn to turn. Running the agent yourself, as above, is how the integration tests exercise it and how you script one-off runs — the developer loop those front-ends wrap is next, in [2.6. Dev Loop](./2.6. Dev Loop.md).

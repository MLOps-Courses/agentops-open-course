# 4.1. Linting

## Why lint an agent's code?

Linters catch the bugs a type checker misses and enforce one consistent style, so reviews argue about behavior, not formatting. For an agent this matters twice over: the tool functions are **prompt material** (their names, docstrings, and structure are read by the model and by ADK's schema generation), and the write actions touch state, so dead code, a swallowed error, or an unsafe SQL string is a real risk. The Python track runs a linter in `mise run check` and a formatter in `mise run format`, and lefthook runs them on every commit.

## How do I lint and format the Python agent?

With **ruff** — one fast tool for both linting and formatting. The linter runs a broad, curated rule set selected in `pyproject.toml`; a few of the enabled families:

```toml
# agents/python/pyproject.toml
[tool.ruff.lint]
select = [
  "B", # flake8-bugbear
  "E", # pycodestyle errors
  "F", # pyflakes
  "I", # isort
  "PTH", # flake8-use-pathlib
  "RUF", # ruff-specific rules
  "S", # flake8-bandit
  "SIM", # flake8-simplify
  "UP", # pyupgrade
  # ...and more: ARG, ASYNC, C4, N, PERF, PT, RET, TID
]
```

Note `S` (flake8-bandit) is a **security** rule set — it flags things like unparameterized SQL — and `PT` enforces idiomatic pytest style. Tests relax a couple of rules where they are noise rather than signal:

```toml
# agents/python/pyproject.toml
[tool.ruff.lint.per-file-ignores]
"tests/**" = [
  "S101", # assert
  "T201", # print statement allowed in tests
]
```

Run them:

```bash
mise run format       # ruff check --select=I --fix  &&  ruff format
mise run check:lint   # uv run ruff check --force-exclude .
```

Any suppression follows one pattern: **specific and documented**, never a blanket `# noqa` — a disabled check should always say why.

## What formats the config and docs?

**dprint** — the single formatter for the non-code files: JSON, TOML, YAML, and Markdown (including these course pages). It is configured at the repo root in `dprint.json` with a shared line width and one wasm plugin per language:

```jsonc
// dprint.json (repo root) — formats *.json, *.toml, *.yaml, *.md
{
  "lineWidth": 120,
  "markdown": { "textWrap": "never" },
  "plugins": [
    "https://plugins.dprint.dev/dockerfile-0.4.1.wasm",
    "https://plugins.dprint.dev/g-plane/pretty_yaml-v0.6.0.wasm",
    "https://plugins.dprint.dev/json-0.22.0.wasm",
    "https://plugins.dprint.dev/markdown-0.22.1.wasm",
    "https://plugins.dprint.dev/toml-0.7.0.wasm"
  ]
}
```

At the root, `mise run format` fans out to dprint plus the Python formatter, and `mise run check` runs `dprint check` alongside the Python linter. lefthook runs the matching formatter on staged files before each commit, so the tree stays clean without anyone remembering to run it.

!!! tip "Format before you check"

    The pre-commit hook is ordered deliberately: formatters run and **restage** first, then
    `check` reads the fixed files. Locally, run `mise run format` before `mise run check` for the
    same effect — a formatting-only diff never fails the lint gate.

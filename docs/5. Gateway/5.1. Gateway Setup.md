# 5.1. Gateway Setup

## What are the prerequisites?

Complete Chapters 1-4, install Ollama, and run the offline gate:

```bash
mise run check
mise run test
ollama pull qwen3:4b
```

The host profile needs no Kubernetes cluster, cloud account, or provider key.

## How do you start the host data plane?

Keep each process in a separate terminal so its logs remain visible.

Start the MCP server:

```bash
cd agents/python
mise run mcp:http
```

Start the A2A agent with both governed client routes:

```bash
cd agents/python
AGENT_GATEWAY_ENABLED=true \
AGENT_MODEL=qwen3:4b \
AGENT_MCP_URL=http://localhost:3000/mcp \
OPENAI_BASE_URL=http://localhost:4000/v1 \
OPENAI_API_KEY=agentgateway \
mise run a2a
```

Start agentgateway from `infra/`:

```bash
agentgateway -f agentgateway/host/config.yaml
```

If Ollama is not already managed as a service, run `ollama serve` in another terminal before submitting a model request.

## How do you verify the listeners?

```bash
curl -fsS http://localhost:3001/.well-known/agent-card.json | jq '{name,url}'
curl -fsS http://localhost:15020/metrics | head
```

The first response should name `Ops Copilot`; the second should return Prometheus text. MCP and model listeners use protocol requests shown on the next pages, so a successful TCP connection alone is not their checkpoint.

## How do you enable host observability?

Start the Compose stack **before** the gateway/agent when you want complete startup traces:

```bash
cd infra
docker compose -f observability/compose.yaml up --build -d
```

Add this to the agent process environment:

```bash
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318
OTEL_EXPORTER_OTLP_PROTOCOL=http/protobuf
OTEL_SERVICE_NAME=ops-copilot
```

The host agent exports traces when those variables are set. The host gateway always exposes metrics on `:15020`, which Compose Prometheus scrapes, but deliberately leaves OTLP export disabled so the optional collector can be stopped without retry noise. Kubernetes gateway profiles enable OTLP because their collector is deployed with them. MLflow is on `localhost:5000`, Grafana on `localhost:3002`, and Prometheus on `localhost:9090`.

## How do you stop the host profile?

Stop the foreground agentgateway, A2A, MCP, and Ollama processes with `Ctrl-C`. Preserve observability data with:

```bash
cd infra
docker compose -f observability/compose.yaml down
```

Add `-v` only when you intentionally want to delete its MLflow, Prometheus, and Grafana volumes.

## What is the setup checkpoint?

Confirm the agent card only through `:3001`, list MCP tools through `:3000`, and make one model request through `:4000`. Do not bypass the gateway ports during the remaining chapter checks.

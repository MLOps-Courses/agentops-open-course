# 8.5. Contributions

## How can I contribute?

The course and its agent are open source, and contributions are welcome — a fixed typo, a clearer explanation, a new tool, a bug fix. Two entry points:

- **[Open an issue](https://github.com/MLOps-Courses/agentops-open-course/issues)** to report a problem, ask a question, or propose a change before you build it.
- **[Submit a pull request](https://github.com/MLOps-Courses/agentops-open-course/pulls)** with the change itself.

The flow is the standard open-source [forking workflow](https://docs.github.com/en/get-started/using-github/github-flow): fork the repo, branch, commit, push to your fork, and open a PR against `main`. Keep `main` deployable and each PR focused on one thing.

## What quality gates run before I commit?

The repository installs [lefthook](https://lefthook.dev/) Git hooks (via `mise run install`), so the checks run automatically — you cannot forget them. Every hook delegates to a `mise run` task, defined in [`lefthook.yml`](https://github.com/MLOps-Courses/agentops-open-course/blob/main/lefthook.yml):

- **On `commit`** — dprint formats your JSON/Markdown/TOML/YAML and restages it, ruff formats the touched Python, then `mise run check` runs, `actionlint` validates any workflow you changed, and `gitleaks` scans for secrets.
- **On `push`** — `mise run test` runs the agent test suite.

Formatters run first and restage the files, so a formatting nit fixes itself instead of failing your commit. If a real check fails, the commit or push is blocked until you fix the cause — never by weakening the check.

## What does CI run?

The **same tasks**, so a green machine means a green pull request. Three GitHub Actions workflows mirror the local hooks:

1. **[`ci.yml`](https://github.com/MLOps-Courses/agentops-open-course/blob/main/.github/workflows/ci.yml)** runs two jobs in parallel: a `format` job (`dprint check` + `actionlint`) plus a `python` job that runs `mise run install`, `mise run check`, and `mise run test`.
1. **[`scan.yml`](https://github.com/MLOps-Courses/agentops-open-course/blob/main/.github/workflows/scan.yml)** runs `mise run scan` (gitleaks over full history) and a Trivy filesystem scan for vulnerabilities, secrets, and misconfigurations.
1. **[`docs.yml`](https://github.com/MLOps-Courses/agentops-open-course/blob/main/.github/workflows/docs.yml)** builds and publishes the site on merge to `main`.

!!! tip "Run the gates locally first"

    Because hooks and CI call identical `mise run` tasks, you can reproduce a red build on your machine: `mise run format && mise run check && mise run test`. Fix it locally, and CI passes on the first try.

## What is the definition of done?

Straight from the repository's own [`AGENTS.md`](https://github.com/MLOps-Courses/agentops-open-course/blob/main/AGENTS.md):

> Before committing: `mise run format && mise run check && mise run test` must pass warning-free.

"Warning-free" is the bar — surface and fix failing tests, lint errors, and broken builds rather than suppressing them.

## How do I write commits and PRs?

Use **[Conventional Commits](https://www.conventionalcommits.org/)** — a typed prefix on each commit subject:

- `feat:` a new capability, `fix:` a bug fix, `docs:` documentation, `chore:` tooling/maintenance, `refactor:` a behavior-preserving change.

This is not just style: the changelog is generated from these prefixes (see [8.2. Releases](./8.2. Releases.md)), so a well-typed history produces good release notes for free.

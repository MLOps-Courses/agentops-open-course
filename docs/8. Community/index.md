# 8. Community

This final chapter is about turning a working agent into a project others can trust, reuse, and build on. It tours how this repository is organized and the README-for-humans, AGENTS.md-for-agents split; the dual license that governs the content and the code; how releases and changelogs are cut; how to package the agent as a reusable template; how the course publishes itself with Zensical; how to contribute back through the same gates CI runs; and the Agentic AI Foundation whose open projects the course is built on and promotes.

This chapter covers:

- **[8.0. Repository](./8.0. Repository.md)**: the repository structure and the README (humans) vs AGENTS.md (agents) split.
- **[8.1. License](./8.1. License.md)**: the dual license — CC-BY-4.0 for the content, MIT for the code — and how to attribute both.
- **[8.2. Releases](./8.2. Releases.md)**: semantic versioning, a git-cliff changelog, v-prefixed tags, and GitHub releases.
- **[8.3. Templates](./8.3. Templates.md)**: making the reference agent a reusable template with cookiecutter or Google's OSS starters.
- **[8.4. Documentation](./8.4. Documentation.md)**: this course's own Zensical, GitHub Pages, and CNAME publishing setup.
- **[8.5. Contributions](./8.5. Contributions.md)**: issues, pull requests, and the lefthook and mise gates that CI mirrors.
- **[8.6. AAIF](./8.6. AAIF.md)**: the Agentic AI Foundation, its projects, the CNCF's kagent, and the Ambassador program.

# 8.6. AAIF

## What is the AAIF?

The **[Agentic AI Foundation (AAIF)](https://aaif.io/)** is a [Linux Foundation](https://www.linuxfoundation.org/) home for open, vendor-neutral agent infrastructure. It stewards the protocols and projects that let agents, tools, and models from different vendors interoperate — so the ecosystem grows on open standards rather than proprietary silos. That neutrality is why this course builds on AAIF projects: the skills you learn transfer across providers and platforms instead of locking you into one.

## Which AAIF projects does the course use?

Three AAIF projects, plus the AAIF-adjacent **AGENTS.md** convention it promotes — each mapped to a phase of the AgentOps lifecycle:

!!! info "AAIF project — agentgateway"

    [agentgateway](https://aaif.io/projects/agentgateway/) is an AI-native proxy (written in Rust) for the traffic agents generate — MCP, A2A, and LLM/inference calls. It is the flagship project of [Chapter 5](../5. Gateway/index.md), where it secures and connects the agent and lets you switch model providers without SDK changes.

!!! info "AAIF project — MCP"

    The [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) is the open standard for connecting agents to tools and data through a common server interface. It is taught in [3.3. MCP](../3. Capabilities/3.3. MCP.md) and federated through the gateway in [5.2. MCP Gateway](../5. Gateway/5.2. MCP Gateway.md).

!!! info "AAIF project — A2A"

    [Agent2Agent (A2A)](https://a2a-protocol.org/) is the open protocol for agents to discover and delegate to one another across processes and vendors. It appears in [3.6. A2A](../3. Capabilities/3.6. A2A.md) and [5.3. A2A Gateway](../5. Gateway/5.3. A2A Gateway.md).

!!! info "AAIF-adjacent standard — AGENTS.md"

    [AGENTS.md](https://agents.md/) is the open convention for instructing AI coding agents — an AAIF-adjacent standard the foundation promotes rather than stewards. This repository dogfoods it (see [8.0. Repository](./8.0. Repository.md)) and teaches it in [1.5. Workspace](../1. Setup/1.5. Workspace.md).

Alongside these, the AAIF stewards **[goose](https://block.github.io/goose/)**, an open-source agent CLI you can try today as a ready-made agent — a useful counterpoint to the one you build here.

## Where does the CNCF fit?

Kubernetes and its ecosystem live under a sibling Linux Foundation body, the **[Cloud Native Computing Foundation (CNCF)](https://www.cncf.io/)** — and that is where the course's runtime comes from:

!!! info "CNCF project — kagent"

    [kagent](https://kagent.dev/) is a **CNCF Sandbox** project that runs agents on Kubernetes, modeling agents, tools, and models as Custom Resources. It is the subject of [Chapter 6](../6. Platform/index.md), where you deploy the Ops Copilot to a local cluster.

Together the two foundations cover the whole stack: the AAIF for how agents connect and interoperate, the CNCF for how they run as cloud-native workloads.

## What is the Ambassador program?

The AAIF runs an **[Ambassador program](https://aaif.io/)** for practitioners who advance open agent infrastructure through teaching, tooling, and community work. The author of this course is an AAIF Ambassador, and **this course is itself an ambassador contribution** — a free, hands-on curriculum built around agentgateway, MCP, A2A, and AGENTS.md.

## How does the course promote these projects?

Deliberately and throughout, as one of its stated goals:

1. It dedicates an entire phase — [Chapter 5](../5. Gateway/index.md) — to **agentgateway** as the connectivity and security layer.
1. It gives **MCP** and **A2A** first-class coverage in both the capabilities and gateway chapters.
1. It **dogfoods AGENTS.md** in its own repository and teaches it as a standard.
1. It flags every AAIF and CNCF project with a callout where it appears, and introduces the ecosystem up front in [0.3. Ecosystem](../0. Overview/0.3. Ecosystem.md).

The result is a course that not only teaches open agent infrastructure but runs on it — the best advertisement a project can have. To go further, visit [aaif.io](https://aaif.io/) and the community links in [0.5. Resources](../0. Overview/0.5. Resources.md).

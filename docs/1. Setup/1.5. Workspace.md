# 1.5. Workspace

## What ties the workflow together?

Three familiar tools, plus one standard this course dogfoods:

- **[Git](https://git-scm.com/)** — version control for everything you build.
- **[GitHub](https://github.com/)** — where the course repository lives and where CI runs.
- **[VS Code](https://code.visualstudio.com/)** — the recommended editor, with first-class terminal, Git, and Python support.
- **[AGENTS.md](https://agents.md/)** — the open standard for instructing AI coding agents, used _inside this repo_.

You can use any editor, but the examples assume a terminal and Git are one keystroke away.

## How do I get the repository?

Clone it and run the one-shot setup from [1.0. System](./1.0. System.md):

```bash
git clone https://github.com/MLOps-Courses/agentops-open-course
cd agentops-open-course
mise install       # provision the pinned tools
mise run install   # docs deps, git hooks, and the agent
```

## What is AGENTS.md and why does it matter?

`AGENTS.md` is a plain-Markdown file at the repository root that tells AI coding agents how to work in the project — the layout, the commands, the conventions, and the definition of done. It is to agents what `README.md` is to humans. This repository practises what it teaches; its own `AGENTS.md` opens with:

> Guidance for AI coding agents working in this repository. This course dogfoods the [AGENTS.md](https://agents.md) standard it teaches. Humans should read `README.md`.

Because agents read one shared file, they follow the same rules the maintainers do — for example that every command routes through `mise run`.

!!! info "AGENTS.md — an open, AAIF-adjacent standard"

    `AGENTS.md` is a vendor-neutral convention adopted across many AI coding tools, and part of the open-standards ecosystem the [AAIF](https://aaif.io/) promotes alongside MCP and A2A. You will meet it again in [8. Community](../8. Community/), where the course covers repository hygiene end to end.

## How does the repo keep quality consistent?

Quality is enforced automatically, not by discipline alone:

- **[dprint](https://dprint.dev/)** formats JSON, Markdown, TOML, and YAML from a single `dprint.json`, so config and docs stay tidy.
- **[lefthook](https://lefthook.dev/)** installs Git hooks (via `mise run install`) that run formatting, linting, and secret scanning on **commit** and the test suites on **push**.
- Every hook delegates to a **`mise run`** task, so the checks you run locally are byte-for-byte the checks CI runs — no surprises in a pull request.

The result: broken formatting, lint errors, or a leaked secret are caught on your machine before they ever reach GitHub.

## What VS Code setup is recommended?

Open the repository folder in VS Code and add the [Python](https://marketplace.visualstudio.com/items?itemName=ms-python.python) and [Ruff](https://marketplace.visualstudio.com/items?itemName=charliermarsh.ruff) extensions. With the toolchain from this chapter in place, you are ready to build your first agent in [Chapter 2](../2. Agents/).

# 1.0. System

## What do I need before starting?

Not much. This chapter assumes you can already:

- Use a **terminal** and run commands.
- Read basic **Python** _or_ **Go** (you only need one — pick a track).
- Use **Git** for version control.

Everything else — the language toolchains, containers, Kubernetes, and the agent framework — is installed and pinned for you in this chapter. No prior agent, Kubernetes, or LLM experience is assumed.

## Which operating systems are supported?

The course works on any Unix-like environment:

- **macOS** — use the built-in Terminal (or [iTerm2](https://iterm2.com/)); install the [Homebrew](https://brew.sh/) package manager.
- **Linux** — any modern distribution with your usual package manager (`apt`, `dnf`, `pacman`, …).
- **Windows** — use the [Windows Subsystem for Linux (WSL2)](https://learn.microsoft.com/windows/wsl/install) and run everything inside an Ubuntu shell, not native PowerShell.

!!! tip "Pick a good shell"

    Any POSIX shell works (`bash`, `zsh`, `fish`). The examples in this course are plain shell commands, so they run unchanged across all three.

## What is mise and why does this course use it?

[mise](https://mise.jdx.dev/) is a single **tool-version manager** and **task runner**. Instead of installing `uv`, `go`, and friends by hand and hoping everyone has the same versions, the repository declares them once in `mise.toml`, and mise installs exactly those versions:

```toml
# mise.toml (repository root)
[tools]
uv = "latest"
dprint = "latest"
go = "1.26"
lefthook = "latest"
gitleaks = "latest"
actionlint = "latest"
```

mise also loads the repository's `.env` for every task, so your provider key is available everywhere without extra setup:

```toml
[[env]]
_.file = ".env" # load the global .env (GOOGLE_API_KEY, AGENT_MODEL) for every task, repo-wide
```

The same `mise run` tasks are what the git hooks and CI call, so your local commands and the pipeline never drift apart.

## How do I install mise?

Install the binary, then activate it in your shell:

```bash
curl https://mise.run | sh              # install mise into ~/.local/bin
echo 'eval "$(mise activate bash)"' >> ~/.bashrc   # bash; use ~/.zshrc for zsh
exec $SHELL                             # reload the shell
mise --version                          # confirm it is on your PATH
```

For `fish`, add `mise activate fish | source` to `~/.config/fish/config.fish` instead. See the [mise docs](https://mise.jdx.dev/getting-started.html) for other shells and installers.

## How do I get the toolchain installed?

Clone the course repository, then let mise install every pinned tool:

```bash
git clone https://github.com/MLOps-Courses/agentops-open-course
cd agentops-open-course
mise install   # installs uv, dprint, go 1.26, lefthook, gitleaks, actionlint
```

`mise install` reads `mise.toml` and provisions the exact versions listed there. The repo deliberately turns off silent auto-install (`run_auto_install = false`) so a missing tool fails loudly instead of being installed behind your back — run `mise install` yourself first.

## What sets up the whole project at once?

Once the tools exist, one task wires everything together — documentation dependencies, git hooks, and **both** agent tracks:

```bash
mise run install
```

That single task runs `uv sync`, `lefthook install`, and then `mise run install` inside `agents/python` and `agents/go`. You do not have to remember the sub-steps. From here, continue to your language track — [1.1. Python](./1.1. Python.md) or [1.2. Go](./1.2. Go.md).

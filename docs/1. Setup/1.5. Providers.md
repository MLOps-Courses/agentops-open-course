# 1.5. Providers

## What do I need to run the agent?

A **model provider**. Both tracks start on **native Gemini** in [Chapters 2–4](../2. Agents/) — the simplest, most idiomatic ADK setup, with an identical mental model in Python and Go. You supply one API key and a model name; that is all Chapter 2 needs. Other providers (OpenAI, Anthropic, local Ollama, …) arrive later through [agentgateway](../5. Gateway/) with zero SDK changes — the course does **not** use LiteLLM.

## How do I configure my key?

The repository ships a template. Copy it to a gitignored `.env` and fill in your key:

```bash
cp .env.example .env
```

```bash
# .env.example
# Copy to `.env` (gitignored) and fill in. mise loads it across the whole repo.
# Native Gemini (Chapters 2-4). Create a key at https://ai.google.dev
GOOGLE_API_KEY=

# The default model for both agent tracks.
AGENT_MODEL=gemini-3.5-flash

# Route to Vertex AI ("global"). Required for Vertex AI express keys; also enables ADC + project mode.
# GOOGLE_GENAI_USE_VERTEXAI=true

# Where the bundled Ops Copilot dataset lives (default: the repo's agents/data).
# AGENT_DATA_DIR=/absolute/path/to/agents/data
```

The root `mise.toml` loads `.env` for every task repo-wide (`[[env]] _.file = ".env"`), so the same key reaches both tracks and all `mise run` commands.

!!! warning "Never commit your key"

    `.env` is gitignored and must stay that way. The `mise run scan` task runs [gitleaks](https://github.com/gitleaks/gitleaks) over the git history to catch a leaked secret before it ships.

## Which key do I need — AI Studio or Vertex express?

Google issues two kinds of key, and they route differently. Read the prefix of your key:

- **AI Studio** key (starts with `AIza…`) — the direct Gemini API. Set `GOOGLE_API_KEY` and **leave `GOOGLE_GENAI_USE_VERTEXAI` unset** (commented out). Create one at [ai.google.dev](https://ai.google.dev/).
- **Vertex AI express** key (starts with `AQ.…`) — routes through Vertex AI. Set `GOOGLE_API_KEY` **and** `GOOGLE_GENAI_USE_VERTEXAI=true`; with an express key the direct Gemini API is blocked, so this flag is required.

Setting `GOOGLE_GENAI_USE_VERTEXAI=true` also enables Application Default Credentials (ADC) and project mode if you prefer credentials over a key.

## Which model does the agent use?

`gemini-3.5-flash`, the default declared for both tracks. It is set explicitly (via `AGENT_MODEL`) because ADK defaults churn — override that one variable to try another Gemini model. The agent code never hardcodes the model beyond a safe default.

## How does each track read the `.env`?

Both tracks read the **same** environment, so switching languages changes no configuration. Application settings use the `AGENT_` prefix (`AGENT_MODEL`, `AGENT_DATA_DIR`); the `GOOGLE_*` variables are consumed directly by the Google GenAI SDK.

=== "Python"

    ```python
    # agents/python/src/agent/config.py
    """Typed, fail-fast configuration for the agent (loaded from the environment / .env)."""

    from __future__ import annotations

    from pathlib import Path

    from pydantic_settings import BaseSettings, SettingsConfigDict

    # The bundled dataset lives at agents/data (shared by both tracks). This file is
    # agents/python/src/agent/config.py, so the repo's agents/ dir is three parents up.
    _DEFAULT_DATA_DIR = Path(__file__).resolve().parents[3] / "data"


    class Settings(BaseSettings):
        """Agent settings. Native Gemini in Ch. 2-4; other providers arrive via agentgateway in Ch. 5."""

        model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8", env_prefix="AGENT_", extra="ignore")

        # Set an explicit model — ADK defaults churn. Override with AGENT_MODEL.
        model: str = "gemini-3.5-flash"

        # Where the bundled Ops Copilot dataset lives. Override with AGENT_DATA_DIR (e.g. in containers).
        data_dir: Path = _DEFAULT_DATA_DIR


    settings = Settings()
    ```

=== "Go"

    ```go
    // agents/go/internal/config/config.go
    // Package config holds the agent's typed, fail-fast configuration, sourced from the environment.
    package config

    import (
    	"os"
    	"path/filepath"
    	"runtime"
    )

    // DefaultModel is set explicitly because ADK model defaults churn.
    // Native Gemini in Ch. 2-4; other providers arrive via agentgateway in Ch. 5.
    const DefaultModel = "gemini-3.5-flash"

    // Config is the agent configuration.
    type Config struct {
    	Model   string // LLM identifier.
    	APIKey  string // GOOGLE_API_KEY for the native Gemini API.
    	DataDir string // Directory holding the bundled Ops Copilot dataset (incidents.db, runbooks/).
    }

    // Load reads configuration from the environment, applying defaults.
    func Load() Config {
    	model := os.Getenv("AGENT_MODEL")
    	if model == "" {
    		model = DefaultModel
    	}
    	return Config{
    		Model:   model,
    		APIKey:  os.Getenv("GOOGLE_API_KEY"),
    		DataDir: dataDir(),
    	}
    }
    ```

Both parse untrusted environment strings into a small, typed configuration at the boundary and fall back to the same explicit defaults — a pattern the course reuses everywhere.

## What about other providers?

They come in [Chapter 5](../5. Gateway/). Instead of wiring an SDK per provider, you point the agent's model endpoint at **agentgateway's LLM backend** and switch providers — including a local [Ollama](https://ollama.com/) — with one endpoint change, symmetrically across Python and Go. Until then, native Gemini is all you need.

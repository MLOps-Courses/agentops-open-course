# 1.3. Kubernetes

## Why does an AgentOps course use Kubernetes?

Kubernetes makes workload identity, configuration, health, persistence, scheduling, and controlled rollout explicit. kagent adds an agent-specific custom resource, while the underlying container remains runnable without the operator.

The course uses Kubernetes only after the host-level agent, tests, and gateway concepts are clear. It is not required for Chapters 0-4.

## Why use k3d locally?

[k3d](https://k3d.io/) runs lightweight k3s nodes inside containers. It is fast to create and delete and supports a local registry, which matches the image-push workflow used by Skaffold. The cluster is named `local`, producing the kubectl context `k3d-local`.

## Which tools are required?

The root `mise.toml` pins k3d, kubectl, Helm, Helmfile, Skaffold, Kustomize, kubeconform, and kube-linter. Install them with `mise install`; use the container engine verified in [1.2](./1.2.%20Containers.md).

## How do you create the registry and cluster?

```bash
cd infra
k3d cluster create --config k3d.yaml
```

The tracked config creates the `local` cluster and `registry.localhost:5050`, disables the load balancer/Traefik, binds the Kubernetes API to loopback, waits for readiness, and switches kubectl to `k3d-local`. The root task `mise run cluster:start` wraps the same command and resumes the cluster if it already exists. Reuse this one cluster and isolate projects by namespace instead of creating a cluster for each chapter.

## How do you verify the cluster checkpoint?

```bash
kubectl cluster-info
kubectl get nodes
k3d registry list
```

Expected state: the current context is `k3d-local`, at least one node is `Ready`, and `registry.localhost` exposes host port `5050`.

If the node remains `NotReady`, inspect `kubectl get pods -A` and `k3d cluster list` before installing kagent.

## How do you stop or delete the shared cluster?

The `local` cluster is shared by projects on this workstation. Stopping it preserves state but interrupts every local workload, so it is not a routine course cleanup command:

```bash
k3d cluster stop local
k3d cluster start local
```

Delete it only when it was created as a dedicated lab and you have confirmed that no other project uses it. Deletion is irreversible for all in-cluster data:

```bash
k3d cluster delete local
```

Chapter 6 provides component-level teardown that keeps the shared cluster intact.

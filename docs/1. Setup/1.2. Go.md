# 1.2. Go

_This is a **Go** track page. Reading the Python track instead? Go back to [1.1. Python](./1.1. Python.md)._

## Why build the agent in Go?

Go gives you static typing, a single self-contained binary, strong performance, and a runtime that is native to Kubernetes and Cloud Run — a natural fit for an _AgentOps_ course that ships and operates agents. The trade-off is a younger ecosystem: the Go [ADK](https://adk.dev/) has no code-execution or planner modules yet. The course teaches the shared 80/20 core and flags the gaps where they land.

## Which Go version does the course use?

Go **1.26**, pinned by mise for the track and declared in the module file:

```toml
# agents/go/mise.toml
[tools]
go = "1.26"
```

```go
// agents/go/go.mod
module github.com/MLOps-Courses/agentops-open-course/agents/go

go 1.26.0
```

## What is a Go module and what is in go.mod?

A **module** is Go's unit of dependency management, declared by `go.mod`: it names the module import path, the Go version, and the required dependencies with exact versions. The reference agent's key requirements are the ADK and the Gemini SDK:

```go
// agents/go/go.mod (excerpt)
require (
	google.golang.org/adk/v2 v2.0.0
	google.golang.org/genai v1.62.0
	modernc.org/sqlite v1.53.0
)
```

`google.golang.org/adk/v2` is the **official** Google ADK for Go (note: `github.com/go-a2a/adk-go` is an unrelated community project — do not use it). `modernc.org/sqlite` is a pure-Go SQLite driver, so the bundled `incidents.db` dataset needs no C toolchain.

## How do I set up the Go track?

From the repository root (after `mise install` from [1.0. System](./1.0. System.md)):

```bash
cd agents/go
mise run install   # runs `go mod tidy`
```

`go mod tidy` resolves and downloads every dependency and prunes anything unused, leaving `go.mod` and `go.sum` consistent.

## Where do the linters and test tools come from?

They are declared as **`go tool` dependencies** in `go.mod`, so they are versioned with the project and installed by the same `go mod tidy` — no separate global installs:

```go
// agents/go/go.mod
tool (
	github.com/golangci/golangci-lint/v2/cmd/golangci-lint
	golang.org/x/tools/cmd/goimports
	golang.org/x/vuln/cmd/govulncheck
	gotest.tools/gotestsum
	mvdan.cc/gofumpt
)
```

## What are the everyday commands?

```bash
mise run format        # goimports + gofumpt
mise run check         # build, golangci-lint, govulncheck (in parallel)
mise run test          # run tests with gotestsum
mise run build         # compile the agent binary to bin/agent
mise run run -- web    # run the agent (needs a provider key); e.g. the web launcher
```

The `run` task passes arguments through to the agent's launcher, so `mise run run -- web` starts the web interface. It needs a model provider — set that up in [1.5. Providers](./1.5. Providers.md) before [Chapter 2](../2. Agents/).

# 1.2. Containers

## Why do agents need containers?

An agent is more than a prompt: it is code, a pinned runtime, dependencies, and (in this course) a bundled dataset. A **container** packages all of that into one immutable image that runs the same on your laptop, in CI, and on Kubernetes. Because [Chapter 6](../6. Platform/) deploys the Ops Copilot to a cluster, you build its image once and ship that exact artifact everywhere — no "works on my machine".

You do not build any images in this chapter. The goal here is only to have a working container runtime so you are ready when the platform chapters need it.

## Docker or Podman?

Either works — they share the same image format and a nearly identical CLI.

- **[Docker](https://www.docker.com/)** — the most common runtime; Docker Desktop bundles everything on macOS and Windows.
- **[Podman](https://podman.io/)** — a daemonless, rootless, fully open-source alternative; `alias docker=podman` covers most commands.

This course is vendor-neutral; pick whichever your platform already provides.

## How do I check my runtime is ready?

Confirm the client and engine respond:

```bash
docker version     # or: podman version
docker run --rm hello-world
```

If `hello-world` prints its greeting, your runtime can pull and run images.

## How will this course build agent images?

That is [Chapter 6.1](../6. Platform/), but the plan is worth knowing now — the course builds **minimal, secure** images with a multi-stage build onto a slim, "distroless-spirit" base (`python:3.13-slim`, since distroless still tracks 3.11), carrying only the app, its runtime, and the dataset.

Small, minimal images reduce the attack surface and start faster — both matter when an agent scales up on Kubernetes.

## Do I need to pull anything now?

No, but you can warm the cache for a component you meet in [Chapter 5](../5. Gateway/). The [agentgateway](https://aaif.io/projects/agentgateway/) image ships from its own registry:

```bash
docker pull cr.agentgateway.dev/agentgateway
```

!!! note "Canonical registry"

    agentgateway is published to `cr.agentgateway.dev/agentgateway`, not Docker Hub or GHCR. Always pin an image tag in production manifests.

# 1.1. Python

## Why build the agent in Python?

Python has the largest agent ecosystem and the most complete [Google ADK](https://adk.dev/) implementation: rubric-based evaluations and features such as code executors and planners, all driven from a standalone `adk` CLI, plus the ADK web developer UI. It is the fastest way to prototype and iterate. If you are new to agents, start here.

## Which Python version does the course use?

Python **3.13**, pinned in two places so there is no ambiguity:

```text
# agents/python/.python-version
3.13
```

```toml
# agents/python/pyproject.toml
requires-python = ">=3.13"
```

Always pin the interpreter per project rather than relying on the system Python — it keeps development, CI, and containers identical.

## What are uv and mise doing here?

Two tools, two jobs:

- **mise** pins the version of `uv` itself (`uv = "latest"` in the root `mise.toml`), so everyone runs the same package manager.
- **[uv](https://docs.astral.sh/uv/)** installs the Python 3.13 interpreter, creates the project's virtual environment (`.venv`), and resolves dependencies from `pyproject.toml` into the locked `uv.lock`.

You never manage a global Python or activate a virtualenv by hand; `uv` does both from the lockfile.

## How do I set up the Python environment?

From the repository root (after `mise install` from [1.0. System](./1.0. System.md)):

```bash
cd agents/python
mise run install   # runs `uv sync`
```

`uv sync` provisions Python 3.13, creates `.venv`, and installs the exact dependencies from `uv.lock`.

## What did that install?

The runtime dependencies of the reference agent:

```toml
# agents/python/pyproject.toml
dependencies = [
  "google-adk>=2.3.0", # native Gemini in Ch. 2–4; other providers via agentgateway in Ch. 5 (no LiteLLM)
  "mcp>=1.16.0", # Model Context Protocol server (FastMCP) + client (McpToolset), Ch. 3.3
  "pydantic-settings>=2.7.1",
  "presidio-analyzer>=2.2.360", # PII detection guardrail (MIT, fully local), Ch. 4.5
  "presidio-anonymizer>=2.2.360", # PII redaction to pair with the analyzer, Ch. 4.5
  "spacy>=3.8,<3.9", # Presidio's NLP engine; pinned to match the en_core_web_sm model below
  "en-core-web-sm", # small English spaCy model for Presidio (URL-pinned in [tool.uv.sources])
  "cryptography>=48.0.1", # security floor: GHSA-537c-gmf6-5ccf (transitive via google-adk); drop once the tree resolves it
]
```

Beyond ADK and MCP, the agent bundles **Presidio** (with a small spaCy model) for local PII redaction ([4.5. Guardrails](../4. Quality/4.5. Guardrails.md)); the `cryptography` floor is a documented security pin ([4.6. Security](../4. Quality/4.6. Security.md)).

`uv sync` also installs the dev group — **ruff** (lint/format), **ty** (type check), **pytest** (tests), and **pip-audit** (dependency vulnerabilities) — so the quality tooling in [Chapter 4](../4. Quality/) works out of the box.

## What are the everyday commands?

Each task is a thin `mise run` wrapper around `uv run`, so you get the project's environment automatically:

```bash
mise run format   # sort imports + format with Ruff
mise run check    # format check, lint, type check, dependency audit
mise run test     # run the pytest suite
mise run web      # launch the ADK developer web UI (needs a provider key)
mise run run      # run the agent in the terminal (needs a provider key)
```

The `web` and `run` tasks need a model provider configured — set that up in [1.4. Providers](./1.4. Providers.md) before you reach [Chapter 2](../2. Agents/).

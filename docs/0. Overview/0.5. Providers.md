# 0.5. Providers

## Which model provider do you start with?

You start with **[Google Gemini](https://ai.google.dev/)**. It is the simplest, most idiomatic setup for Google ADK, and the mental model is identical in Python and Go. Both tracks use **native Gemini** in Chapters 2–4 — no extra proxy, no vendor-neutral abstraction to learn first. You reach other providers later, once the agent works and you have somewhere to switch them (the gateway, in Chapter 5).

Starting on one provider keeps the early chapters focused on _agents_, not on plumbing several SDKs.

## What is the default model?

The course default is **`gemini-3.5-flash`** — a fast, inexpensive model that is more than capable for the Ops Copilot. Flash-class models keep latency and token cost low, which matters because an agent makes many model calls per task.

!!! warning "Always set the model explicitly"

    Provider defaults churn between releases. Pin your model (`model="gemini-3.5-flash"`) rather than relying on a default, so your agent's behavior stays reproducible.

## What kinds of Gemini keys are there?

There are two ways to authenticate to Gemini, and they use different key formats:

1. **AI Studio (direct Gemini API)** — keys from [Google AI Studio](https://aistudio.google.com/) that start with **`AIza…`**. This is the quickest way to get a key. It talks to the `generativelanguage` endpoint.
1. **Vertex AI express** — keys that start with **`AQ.…`**, obtained through [Vertex AI](https://cloud.google.com/vertex-ai) in express mode. You select this path by setting `GOOGLE_GENAI_USE_VERTEXAI=true`.

Beyond keys, you can also use **Application Default Credentials (ADC)** via `gcloud auth
application-default login` for the Vertex path, which avoids putting a key in your `.env` at all. Chapter [1.5. Providers](../1. Setup/1.5. Providers.md) and [2.3. Models](../2. Agents/2.3. Models.md) walk through each option.

!!! note "Keys are not interchangeable"

    An `AIza…` key works only against the direct Gemini API, and an `AQ.…` key works only against Vertex AI. Match the key type to the endpoint (`GOOGLE_GENAI_USE_VERTEXAI`), or you will get authentication errors. Keep every key out of git — in a gitignored `.env`.

## How do you reach other providers?

You reach OpenAI, Anthropic, OpenRouter, and local **[Ollama](https://ollama.com/)** models through **[agentgateway](../5. Gateway/5.4. Model Gateway.md)**, not through the agent SDK. In Chapter 5 you point the agent's model endpoint at the gateway's LLM backend; from then on, switching providers is a **one-line configuration change** with **zero SDK changes**, and it works symmetrically in Python and Go.

!!! info "No LiteLLM"

    This course intentionally does **not** use LiteLLM. Provider neutrality is handled at the gateway — one endpoint, one switch — which keeps the agent code identical across providers and languages, and consolidates auth, rate limiting, and observability in one place.

## Do you need to pay to follow along?

No. Gemini offers a **free tier** that is sufficient to build and evaluate the Ops Copilot, and **Ollama** runs models entirely on your own machine at zero cost once you reach the gateway chapter. The whole course is designed to run local-first, so you can complete it without a meaningful cloud bill.

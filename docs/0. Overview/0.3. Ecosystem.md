# 0.3. Ecosystem

## What does the open-source agent ecosystem look like?

The agent space is young and moving fast, but a coherent open-source stack has emerged. This course builds on four pillars that map to the lifecycle:

- **[Google ADK](https://adk.dev/)** — the framework you build the agent with.
- **[agentgateway](https://aaif.io/projects/agentgateway/)** — the proxy that connects and secures it.
- **[kagent](https://kagent.dev/)** — the runtime that operates it on Kubernetes.
- **Open standards** — [MCP](https://modelcontextprotocol.io/), [A2A](https://a2a-protocol.org/), and [AGENTS.md](https://agents.md/) — the protocols that let these pieces interoperate.

Two foundations steward much of this work: the **AAIF** and the **CNCF**.

## What is Google ADK?

The [Agent Development Kit (ADK)](https://adk.dev/) is Google's open-source framework for building agents. It owns the agentic loop — sessions, tool calls, workflows, memory, and multi-agent orchestration — so you focus on your agent's behavior rather than plumbing. The course uses **ADK 2.0** in two tracks: Python (`google-adk` on PyPI, import `google.adk`) and Go (module `google.golang.org/adk/v2`). ADK 2.0's headline feature is a **graph-based Workflow Runtime** for deterministic, dynamic, and collaborative multi-agent workflows, alongside the classic Sequential/Parallel/Loop workflow agents.

## What is agentgateway?

!!! info "AAIF project"

    [agentgateway](https://aaif.io/projects/agentgateway/) is a project of the **[Agentic AI Foundation (AAIF)](https://aaif.io/)**.

agentgateway is an AI-native proxy (written in Rust) for the traffic agents generate: **MCP**, **A2A**, and **LLM/inference** calls. It was created by Solo.io and donated to the Linux Foundation, and is now an AAIF project. You run it standalone as a single binary (`agentgateway -f config.yaml`) or as the container `cr.agentgateway.dev/agentgateway`, with an admin UI on `localhost:15000`. In this course it does two jobs: it fronts your MCP and A2A traffic with auth, rate limits, and prompt guards ([Chapter 5](../5. Gateway/index.md)), and it lets you switch model providers by pointing the agent at its LLM backend — no SDK changes.

## What is kagent?

!!! info "CNCF project"

    [kagent](https://kagent.dev/) is a **[CNCF](https://www.cncf.io/) Sandbox** project.

kagent brings agents to Kubernetes. It models agents, tools, and model configurations as **Custom Resources** (CRDs under the `kagent.dev` API group) and runs them with a controller, an engine built on ADK, and a web UI. You install it with the `kagent` CLI or its Helm charts. In this course you package the Ops Copilot as a container and deploy it with kagent's `Agent` resource (the `type: BYO` variant that points at your own image), then wire in agentgateway as its data plane ([Chapter 6](../6. Platform/index.md)). kagent is CNCF Sandbox and pre-1.0, so you also learn to pin versions and treat breaking API changes as normal.

## Which open standards matter?

Standards are what keep this stack vendor-neutral. Three matter most:

- **[MCP (Model Context Protocol)](https://modelcontextprotocol.io/)** — an open standard, introduced by Anthropic, for connecting agents to tools and data through a common server interface. Any MCP-compatible tool works with any MCP-compatible agent. Covered in [3.3. MCP](../3. Capabilities/3.3. MCP.md) and [5.2. MCP Gateway](../5. Gateway/5.2. MCP Gateway.md).
- **[A2A (Agent2Agent)](https://a2a-protocol.org/)** — an open protocol, originally from Google, for agents to discover and delegate to one another across processes and vendors. Covered in [3.6. A2A](../3. Capabilities/3.6. A2A.md) and [5.3. A2A Gateway](../5. Gateway/5.3. A2A Gateway.md).
- **[AGENTS.md](https://agents.md/)** — a simple open convention: a Markdown file at the repo root that gives coding agents the instructions and context they need. This repository dogfoods it, and it is taught in [1.6. Workspace](../1. Setup/1.6. Workspace.md).

## What are the AAIF and CNCF?

- The **[Agentic AI Foundation (AAIF)](https://aaif.io/)** is a Linux Foundation home for open, vendor-neutral agent infrastructure. Its projects include agentgateway, MCP, A2A, and the [goose](https://block.github.io/goose/) agent CLI. The author of this course is an [AAIF Ambassador](https://aaif.io/), and the course is itself an ambassador contribution.
- The **[Cloud Native Computing Foundation (CNCF)](https://www.cncf.io/)** is the Linux Foundation home for cloud-native software — Kubernetes, Prometheus, and now kagent among many others. Its Sandbox tier is where early, promising projects like kagent incubate.

Because this stack is built on open standards and neutral foundations, the skills you learn here transfer across providers and platforms rather than locking you into one vendor.

# 0.4. Languages

## Why does this course cover both Python and Go?

Google ADK 2.0 ships in **both Python and Go**, and each is a legitimate choice for production agents. Python dominates the AI ecosystem and offers the richest tooling; Go offers static typing, a single binary, and a Kubernetes-native operational story. Rather than pick for you, the course teaches the same agent in **two parallel tracks** so you can learn the language you will actually ship, and reason about the trade-offs.

The concepts, protocols, and operations are identical across tracks — only the code differs.

## What are Python's strengths and weaknesses?

**Strengths:**

- The **biggest ecosystem** for AI: the widest range of libraries, examples, and community support.
- A **standalone `adk` CLI** (`adk web`, `adk run`, `adk eval`) — a batteries-included command-line workflow. (Both tracks serve the same ADK web developer UI; in Go you reach it through your own compiled binary instead of a separate CLI.)
- **Rubric-based evaluations** and the most mature ADK feature set, including code-execution sandboxes and planner modules.

**Weaknesses:**

- **Rapid, sometimes breaking releases** — you pin versions and re-verify often.
- Dynamic typing means some classes of error surface at run time rather than compile time (mitigated with type checking, taught in [4.0. Typing](../4. Quality/4.0. Typing.md)).

## What are Go's strengths and weaknesses?

**Strengths:**

- **Static typing** catches many errors at compile time.
- **Single self-contained binary** — trivial to containerize and deploy, and a natural fit for Kubernetes and Cloud Run.
- **Performance** and a strong concurrency story for production services.

**Weaknesses:**

- A **younger agent ecosystem** with fewer examples than Python.
- **No code-execution or planner modules** yet, and no `llms.txt` — some advanced Python-only features have no Go equivalent (the course flags these where they land).

## Which language should you choose?

Both reach the same destination. Pick by where the agent will live and what your team already uses:

| Aspect        | Python                                            | Go                                                   |
| ------------- | ------------------------------------------------- | ---------------------------------------------------- |
| Package       | `google-adk` (import `google.adk`)                | `google.golang.org/adk/v2`                           |
| Best when     | you want the richest ecosystem and tooling        | you want a single binary and k8s-native deploys      |
| Standout      | standalone `adk` CLI, rubric evals, most features | static typing, performance, single-binary ops        |
| Watch out for | rapid breaking releases                           | younger ecosystem; no code-execution/planner modules |

If you have no strong preference, **start with Python** for the smoother first experience, then try the Go track once you know the concepts. If you operate on Kubernetes and value typed, compiled services, **Go** will feel more at home.

## How do you follow one track — or both?

Most pages are language-neutral prose with code in **Python / Go tabs**. Click your language tab once and the choice **syncs across the whole site**, so you read a consistent single-language experience. The one exception is the hero build in Chapter 2, which is split into a matched pair of standalone pages — [2.1. First Agent (Python)](../2. Agents/2.1. First Agent (Python).md) and [2.2. First Agent (Go)](../2. Agents/2.2. First Agent (Go).md) — so you can build without tab-switching.

To follow **both** languages, simply read both tabs (or both First-Agent pages). Because the shared concepts carry over, the second track is much faster the second time through.

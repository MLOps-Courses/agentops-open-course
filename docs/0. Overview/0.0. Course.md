# 0.0. Course

## What will this course teach you?

This course teaches you to build a real AI agent end to end and then operate it like production software — the discipline we call _AgentOps_. Instead of stopping at a demo that works once in a notebook, you will learn to:

- Build an agent with [Google ADK 2.0](https://adk.dev/) in **Python**.
- Give it real capabilities: tools, skills, [MCP](https://modelcontextprotocol.io/) servers, memory/RAG, multi-agent workflows, and [A2A](https://a2a-protocol.org/) delegation.
- Make it trustworthy with typing, tests, evaluations, guardrails, and human-in-the-loop.
- Secure and connect it through [agentgateway](https://aaif.io/projects/agentgateway/), then deploy it on Kubernetes with [kagent](https://kagent.dev/).
- Observe it in production: tracing, monitoring, cost tracking, feedback, and governance.

Everything runs **100% open source** and **local-first**, so you can follow along on your own machine without a cloud bill.

## Who is this course for?

It is written for people who can already code and now want to build, ship, and operate agents professionally:

- **Software developers** moving into AI engineering who want the full agent lifecycle, not just prompt tricks.
- **ML/AI engineers** who know models but want production-grade agent patterns and tooling.
- **Platform and DevOps engineers** who will run agents as real workloads on Kubernetes.

No prior experience with agents, LLMs, or Kubernetes is assumed — each is introduced from first principles.

## What will you build?

You build one agent across the whole course: a DevOps **"Ops Copilot"** that helps an on-call engineer triage and resolve incidents for a fictional service. It works entirely from a local, bundled dataset — a small SQLite incident database, a set of Markdown runbooks, and sample logs — so it is fully reproducible and evaluable offline. Every action is mock and local (a "restart" flips a status and appends to an audit log; no real infrastructure is touched).

This single use case is chosen because it exercises every capability the curriculum teaches: function tools, MCP, memory/RAG, workflows, A2A, evaluations, guardrails, the gateway, and Kubernetes deployment. It is also on-brand for an _AgentOps_ course — an agent that does ops, later operated and observed on a real platform.

The course is Chapter 0 (this overview) plus eight chapters, each named after a concrete component:

## What are the prerequisites?

You should be comfortable with:

1. **Python**: basic [Python](https://www.python.org/).
1. **The command line**: installing software, running commands, and navigating the filesystem.
1. **Git**: cloning, committing, and pushing to a repository.

You do **not** need prior experience with agents, LLMs, Kubernetes, or a specific cloud.

## How do you follow the course?

The course is written in Python: prose plus Python code you can run as you read.

The course is self-paced. If you already know tools like Git, containers, or Kubernetes, you will move faster. A pragmatic path is to build a working agent early, then progressively make it right and production-ready — one chapter at a time.

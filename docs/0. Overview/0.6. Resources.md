# 0.6. Resources

## What AI coding assistants should you use?

This is a course about agents, so it makes sense to build it _with_ agents. AI coding assistants — such as [Gemini CLI](https://github.com/google-gemini/gemini-cli), [Cursor](https://cursor.com/), or the AAIF's own [goose](https://block.github.io/goose/) — can scaffold code, run your tests, and explain unfamiliar APIs. Treat them as a fast pair programmer: they accelerate the work, but you stay responsible for reviewing and understanding every change. To steer them well, this repository ships an **[AGENTS.md](https://agents.md/)** file that gives assistants the project's conventions and context.

## What are Agent Skills?

**Agent Skills** are reusable, progressive-disclosure instructions that teach an assistant how to perform a specific task in your project — a small folder of Markdown and scripts (each a `SKILL.md`) that the agent loads only when it is relevant. (Distinct from `AGENTS.md`, which is one always-on file of project conventions.) They keep prompts short while making capabilities discoverable. You learn to author them for your own agent in [3.2. Skills](../3. Capabilities/3.2. Skills.md), and they are a practical way to package the conventions from this course for your team's assistants.

## Where is the community?

- **[Agentic AI Foundation (AAIF)](https://aaif.io/)** — the vendor-neutral home for agentgateway, MCP, A2A, and goose. If you want to contribute to open agent infrastructure, start here; the [AAIF Ambassador program](https://aaif.io/) welcomes practitioners who teach and promote these projects.
- **[goose](https://block.github.io/goose/)** — an open-source AAIF agent CLI. A great "existing agent to try" before and while you build your own.
- **Course repository** — issues and discussions live on [GitHub](https://github.com/MLOps-Courses/agentops-open-course); suggestions and fixes are welcome.

## What do the key terms mean?

A short glossary of terms used throughout the course:

- **Agent** — a program that uses an LLM in a loop to decide and take actions via tools.
- **Tool** — a function an agent can call to observe or change the world (e.g. query a database, restart a service).
- **MCP (Model Context Protocol)** — an open standard for exposing tools and context to agents through a common server interface.
- **A2A (Agent2Agent)** — an open protocol for agents to discover and delegate to one another.
- **RAG (Retrieval-Augmented Generation)** — retrieving relevant knowledge (e.g. runbooks) and feeding it into the model so answers are grounded in your data.
- **HITL (Human-in-the-Loop)** — pausing for a person to approve a risky or irreversible action before the agent proceeds.
- **Eval (Evaluation)** — a repeatable test that judges an agent's behavior (task success, tool accuracy) rather than exact output strings.
- **Runbook** — a documented procedure for handling a situation; here, the Markdown knowledge base the Ops Copilot retrieves from.
- **Guardrail** — a check that validates an agent's inputs, outputs, or actions to keep it safe and within policy.

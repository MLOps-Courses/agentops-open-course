# 3.1. Tools

## What is a function tool?

A **tool** is a function the model can call to observe or change the world. Without tools an agent is just a chatbot; with them it can query a database, read a runbook, or restart a service. In ADK, a tool is an ordinary typed function — you write the function, ADK exposes its signature to the model, and when the model decides to call it, ADK runs it and feeds the result back into the loop.

The Ops Copilot starts with three read-only tools over the bundled `incidents.db`:

- `list_incidents` — list incidents, most recent first, optionally filtered.
- `get_incident` — fetch one incident by id, including its runbook slug.
- `get_service_status` — a service's current status plus its open incidents.

The guarded write actions (`restart_service`, `resolve_incident`) that require human approval arrive in [4.5. Guardrails](../4. Quality/4.5. Guardrails.md); this page is the read-only foundation.

## How do I write a tool?

Write a plain, typed function with a clear description, then register it on the agent. ADK reads the function's shape to build the schema the model sees.

=== "Python"

    ```python
    # agents/python/src/agent/tools.py
    def list_incidents(status: str = "", service: str = "") -> dict[str, Any]:
        """List incidents on the platform, most recent first.

        Args:
            status: Optional filter — one of ``open``, ``investigating``, or ``resolved``.
                Leave empty to list incidents of every status.
            service: Optional service name to filter by (e.g. ``checkout``). Leave empty for all.

        Returns:
            A dict with ``count`` and an ``incidents`` list (id, service, title, severity, status).
        """
        rows = data.list_incidents(status=status or None, service=service or None)
        incidents = [
            {
                "id": row["id"],
                "service": row["service"],
                "title": row["title"],
                "severity": row["severity"],
                "status": row["status"],
            }
            for row in rows
        ]
        return {"count": len(incidents), "incidents": incidents}


    def get_incident(incident_id: str) -> dict[str, Any]:
        """Get the full details of one incident by its id.

        Args:
            incident_id: The incident identifier, e.g. ``INC-001``.

        Returns:
            ``{"incident": {...}}`` with the record (including ``runbook`` and ``summary``),
            or an ``error`` if unknown.
        """
        incident = data.get_incident(incident_id)
        if incident is None:
            return {"error": f"No incident found with id {incident_id!r}."}
        return {"incident": incident}


    def get_service_status(name: str) -> dict[str, Any]:
        """Get the current status of a service and its open incidents.

        Args:
            name: The service name, e.g. ``checkout`` or ``inventory``.

        Returns:
            ``{"service": {...}, "open_incidents": [...]}``, or an ``error`` if unknown.
        """
        service = data.get_service(name)
        if service is None:
            known = ", ".join(row["name"] for row in data.list_services())
            return {"error": f"No service named {name!r}. Known services: {known}."}
        open_incidents = [
            {"id": row["id"], "title": row["title"], "severity": row["severity"], "status": row["status"]}
            for row in data.list_incidents(service=name)
            if row["status"] != "resolved"
        ]
        return {"service": service, "open_incidents": open_incidents}


    # The tools registered on the Ops Copilot. Guarded actions (restart/resolve) join in Ch. 4.5.
    ALL_TOOLS: list[ToolUnion] = [list_incidents, get_incident, get_service_status]
    ```

=== "Go"

    ```go
    // agents/go/internal/tools/tools.go
    // listIncidentsInput filters the incident list; empty fields mean "no filter".
    type listIncidentsInput struct {
    	Status  string `json:"status"`  // one of open, investigating, resolved; empty for all
    	Service string `json:"service"` // service name, e.g. checkout; empty for all
    }

    type listIncidentsOutput struct {
    	Count     int             `json:"count"`
    	Incidents []data.Incident `json:"incidents"`
    }

    // All builds the tools registered on the agent, each closing over the shared data store.
    func All(store *data.Store) ([]tool.Tool, error) {
    	all := make([]tool.Tool, 0, 3)
    	for _, build := range []func(*data.Store) (tool.Tool, error){
    		listIncidentsTool, getIncidentTool, getServiceStatusTool,
    	} {
    		tl, err := build(store)
    		if err != nil {
    			return nil, err
    		}
    		all = append(all, tl)
    	}
    	return all, nil
    }

    func listIncidentsTool(store *data.Store) (tool.Tool, error) {
    	return functiontool.New(functiontool.Config{
    		Name:        "list_incidents",
    		Description: "List incidents on the platform, most recent first. Optionally filter by status and/or service.",
    	}, func(_ agent.Context, in listIncidentsInput) (listIncidentsOutput, error) {
    		incidents, err := store.ListIncidents(in.Status, in.Service)
    		if err != nil {
    			return listIncidentsOutput{}, err
    		}
    		return listIncidentsOutput{Count: len(incidents), Incidents: incidents}, nil
    	})
    }

    func getServiceStatusTool(store *data.Store) (tool.Tool, error) {
    	return functiontool.New(functiontool.Config{
    		Name:        "get_service_status",
    		Description: "Get the current status of a service and its open incidents.",
    	}, func(_ agent.Context, in getServiceStatusInput) (map[string]any, error) {
    		service, ok, err := store.GetService(in.Name)
    		if err != nil {
    			return nil, err
    		}
    		if !ok {
    			return map[string]any{"error": fmt.Sprintf("No service named %q.", in.Name)}, nil
    		}
    		incidents, err := store.ListIncidents("", in.Name)
    		if err != nil {
    			return nil, err
    		}
    		open := []data.Incident{}
    		for _, inc := range incidents {
    			if inc.Status != "resolved" {
    				open = append(open, inc)
    			}
    		}
    		return map[string]any{"service": service, "open_incidents": open}, nil
    	})
    }
    ```

The Python tools are plain functions passed straight into `Agent(tools=[...])`; ADK auto-wraps each as a `FunctionTool`. The Go tools are built with `functiontool.New`, taking an explicit name and description plus a typed handler `func(agent.Context, In) (Out, error)`.

## How does ADK turn a function into a schema the model sees?

The model never sees your code — it sees a **JSON schema** describing each tool's name, purpose, and parameters. ADK generates that schema, and the two languages source it differently:

- **Python** — from **type hints and the Google-style docstring**. The parameter names and annotations become the schema's properties; the docstring summary becomes the tool description and the `Args:` lines document each parameter. That is why the tools carry full docstrings: they are prompt material, not just developer notes.
- **Go** — from the **handler's input/output struct types and their `json` tags**, with the tool's name and description passed explicitly to `functiontool.New`. The `json:"status"` tag names the field the model must fill; the trailing `// one of open, investigating, resolved` comments guide you, while the model relies on the tool `Description` and field names.

In both cases the tool returns a plain map/struct so the result serializes cleanly back into the conversation. Keep names and descriptions precise — they are the contract the model reasons over.

## Where do the tools get their data?

From a separate **data-access layer** (`data.py` / `data.go`), kept apart from the tool functions so it can be unit-tested directly against `agents/data/incidents.db`. The tools stay thin — they shape data-layer results into model-friendly output; the SQL lives one layer down.

=== "Python"

    ```python
    # agents/python/src/agent/data.py
    @contextmanager
    def _connect() -> Iterator[sqlite3.Connection]:
        """Open a read/write connection with row access by column name; always closed."""
        connection = sqlite3.connect(db_path())
        connection.row_factory = sqlite3.Row
        try:
            yield connection
        finally:
            connection.close()


    def list_incidents(status: str | None = None, service: str | None = None) -> list[dict[str, Any]]:
        """Return incidents, newest first, optionally filtered by status and/or service."""
        query = "SELECT * FROM incidents"
        clauses: list[str] = []
        params: list[str] = []
        if status is not None:
            clauses.append("status = ?")
            params.append(status)
        if service is not None:
            clauses.append("service = ?")
            params.append(service)
        if clauses:
            query += " WHERE " + " AND ".join(clauses)
        query += " ORDER BY opened_at DESC"
        with _connect() as connection:
            return [dict(row) for row in connection.execute(query, params)]
    ```

=== "Go"

    ```go
    // agents/go/internal/data/data.go
    // Store owns the database handle and the data directory (for locating runbooks).
    type Store struct {
    	db  *sql.DB
    	dir string
    }

    // A single static query keeps SQL out of string building: an empty filter argument disables
    // its clause, and every user value is bound with a placeholder (no injection surface).
    const listIncidentsQuery = "SELECT " + incidentColumns + " FROM incidents " +
    	"WHERE (? = '' OR status = ?) AND (? = '' OR service = ?) ORDER BY opened_at DESC"

    // ListIncidents returns incidents newest first, optionally filtered by status and/or service
    // (pass "" to skip a filter).
    func (s *Store) ListIncidents(status, service string) ([]Incident, error) {
    	rows, err := s.db.Query(listIncidentsQuery, status, status, service, service)
    	if err != nil {
    		return nil, fmt.Errorf("querying incidents: %w", err)
    	}
    	defer func() { _ = rows.Close() }()

    	incidents := []Incident{}
    	for rows.Next() {
    		inc, err := scanIncident(rows)
    		if err != nil {
    			return nil, err
    		}
    		incidents = append(incidents, inc)
    	}
    	return incidents, rows.Err()
    }
    ```

Both bind every user value with a `?` placeholder — no string concatenation of inputs, so there is no SQL-injection surface even though the model chooses the filter values. In Python the state is a module-level connection helper; in Go it is an explicit `*data.Store` the tools close over, which makes the dependency obvious and easy to test.

## What about OpenAPI toolsets?

ADK can also generate a whole toolset from an **OpenAPI specification** — point it at a spec and every operation becomes a callable tool, no per-endpoint code. This is how you would later wrap a real services API instead of hand-writing tools.

!!! note "Python-only today"

    The OpenAPI toolset ships in the **Python** ADK; the **Go** ADK does not yet have an equivalent. When you need it on the Go track, expose the API through an MCP server ([3.3. MCP](./3.3. MCP.md)) and consume that instead — an approach that works identically in both languages.

For the Ops Copilot the three hand-written function tools are enough, and they keep the course fully offline. Next, teach the agent to fetch procedures on demand with [3.2. Skills](./3.2. Skills.md).

# 3.2. Skills

## What is an Agent Skill?

An **Agent Skill** is a folder of instructions the model can pull in only when it is relevant. Each skill has a `SKILL.md` with a `name` and `description` in its front-matter, then the full procedure below. The model first sees **only** the name and description of every skill; it loads a skill's full body on demand — a pattern called **progressive disclosure**. That keeps the base context small while still giving the agent deep, reusable know-how when it needs it.

Skills are how you encode operational procedure — "how to triage", "how to remediate" — without bloating the system prompt with every playbook at once.

!!! info "The Agent Skills standard"

    Skills follow the open **Agent Skills** convention (a `SKILL.md` with `name`/`description` front-matter, optional `assets/` and `references/`), the same format used by AI coding assistants. Because it is a plain folder-and-Markdown standard, the Ops Copilot's skills live once in `agents/data/skills/` and are shared by **both** the Python and Go tracks.

## What does a SKILL.md look like?

Front-matter for discovery, then Markdown instructions the model follows once the skill is loaded. Here is the Ops Copilot's triage skill:

```text
# agents/data/skills/incident-triage/SKILL.md
---
name: incident-triage
description: How to prioritize and triage open incidents for the Ops Copilot.
---

# Incident Triage Skill

Use this skill when asked to triage, prioritize, or decide what to work on first.

## Instructions

1. List the unresolved incidents with `list_incidents` (status `open` and `investigating`).
1. Rank them by severity — **SEV1** (highest) before SEV2 before SEV3. Break ties by the oldest `opened_at` first (longer-running incidents are more urgent).
1. For the top incident, check the affected service with `get_service_status`; a `down` service outranks a `degraded` one.
1. Report the single most urgent incident first: its id, service, severity, and one-line summary, then briefly list the rest in priority order.
1. Never invent incidents or severities — only report what the tools return.
```

The `description` is the whole cost the skill imposes on the base prompt — one line. Only when the model recognizes a triage request does it expand the instructions and follow them, referencing the same function tools from [3.1. Tools](./3.1. Tools.md). A second skill, `remediation`, does the same for proposing runbook-backed fixes.

## How does the agent load skills?

Point a **skill toolset** at the shared skills directory; it discovers every skill and exposes progressive-disclosure loading to the model. The loader is a few lines in each track.

=== "Python"

    ```python
    # agents/python/src/agent/skills.py
    from pathlib import Path

    from google.adk.skills import list_skills_in_dir, load_skill_from_dir
    from google.adk.tools.skill_toolset import SkillToolset

    from .config import settings


    def skills_dir() -> Path:
        """Return the directory holding the Ops Copilot skills."""
        return settings.data_dir / "skills"


    def skill_toolset() -> SkillToolset:
        """Build a SkillToolset from every skill in the skills directory (progressive disclosure)."""
        base = skills_dir()
        skills = [load_skill_from_dir(base / name) for name in list_skills_in_dir(base)]
        return SkillToolset(skills=skills)
    ```

=== "Go"

    ```go
    // agents/go/internal/skills/skills.go
    import (
    	"context"
    	"fmt"
    	"os"
    	"path/filepath"

    	"google.golang.org/adk/v2/tool"
    	"google.golang.org/adk/v2/tool/skilltoolset"
    	"google.golang.org/adk/v2/tool/skilltoolset/skill"
    )

    // Toolset builds a skill toolset from every skill under <dataDir>/skills.
    func Toolset(ctx context.Context, dataDir string) (tool.Toolset, error) {
    	source := skill.NewFileSystemSource(os.DirFS(filepath.Join(dataDir, "skills")))
    	toolset, err := skilltoolset.New(ctx, skilltoolset.Config{Source: source})
    	if err != nil {
    		return nil, fmt.Errorf("building skill toolset: %w", err)
    	}
    	return toolset, nil
    }
    ```

Both resolve `<data_dir>/skills`, scan it, and hand ADK a toolset. The Python `SkillToolset` takes a list of loaded skills; the Go `skilltoolset.New` takes a filesystem `Source` (`os.DirFS`) that it reads lazily. Add a skill by dropping a new folder with a `SKILL.md` into `agents/data/skills/` — no code change, and both tracks pick it up.

## When should I use a skill instead of a tool or a prompt?

They solve different problems, and good agents combine them:

- **Tool** — an _action_: code the model runs to fetch data or change state (`get_incident`). Use a tool when the agent needs to _do_ something.
- **Skill** — a _procedure_: instructions on _how_ to use the tools for a class of task (how to triage). Use a skill when the agent needs to _know how_.
- **System prompt** — the agent's _always-on_ identity and rules. Keep it short; push situational know-how into skills so it loads only when relevant.

Progressive disclosure is the payoff: you can ship many detailed procedures without paying for all of them on every turn. The model reads a menu of one-line descriptions and expands only what the current task needs.
